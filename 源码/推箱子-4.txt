碰撞实验
当人物移动过程中，遇到了障碍，应该不能够通过障碍，此时我们就要考虑所有模型
之间的碰撞问题，也就是我们所说的碰撞实验
模型碰撞思想
1.人物在屏幕上显示，显示在哪里？程序怎么知道人物在什么位置？
现实模型：车票售票大厅，售票人员，关心旅客有没有身份证，旅客有没有钱，旅客买票从哪里到哪里？
2.什么是障碍？计算机怎么知道树木就是障碍的？羊是不是障碍？
二维数组中的数据1代表障碍,0代表空地
我们设计数据2代表人物，人物的数据在原始的二维数组中是不存在的，因此我们把
人物添加到程序中时，必须把这个数据2放到二维数组中
2.1代码应该写在什么位置？
应该把这个数据2的修改放入到人物创建的方法中
放在名称为wolfInit的方法中，也就是人物初始化方法
2.2代码应该怎么写？
将数据2放到灰太狼对应的位置上面
思考：人物在移动的过程中，这个数据2的位置在不停的改变
我想找一个地方，写上现在人物在哪里？
设计一个值，用来描述人物现在在第几行
在数组的下面添加
int wx;
设计一个值，用来描述人物现在在第几列
在数组的下面添加
int wy;
在人物初始化的方法第一句写上
wx = 6;
wy = 4;
把人物的初始化坐标位置，改成由x与y计算而来
lab_wolf.setBounds(12 + wx*50, 36 + wy*50, 50, 50);

人物的移动会导致这个wx与wy的变化，具体的变化规则是
移动控制坐标改变的代码写在对应的移动事件中
右移：wx = wx+1;
左移：wx = wx-1;
上移：wy = wy-1;
下移：wy = wy+1;

3.当人物遇到障碍后，怎么办？障碍针对人物的位置不同，是否会对人物的移动产生
影响
树木是障碍，但是这只是用户看到的结果，实际上树木在程序中以数字1的格式出现
所谓的遇到障碍，就是判断一下我下面要移动过去的位置上面是不是1？
模型判断的误区
datas[10][1]代表的是第10行，第一列
datas[x][y]
x描述的是列
y描述的是行
控制人物遇到障碍后，不做任何事情，将下列代码写入到对应的移动事件中
当前代码是向右移动控制的代码，如果是向其他方向，需要根据情况修改里面的
wy与wx位置上的数据变化
if(datas[wy][wx+1] == 1){
	return;
}


碰撞模型设计
碰撞模型是通过数组中的数据进行检测的，不同的数据碰撞检测不同

人物模型移动设计
人物模型移动是通过两个整数来模拟当前人物在数组中的位置
同时也是将屏幕划分成若干个小的方格，然后人物在这些小方格间跳动

碰撞模型实现
对不可通过的数据进行判断，比如说树木对应的数据1，不可通过，那么就可以使用
判断的形式提前结束移动操作
